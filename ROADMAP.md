# Roadmap

This roadmap is intentionally staged. StepSpark should not skip foundational work in order to ship a fragile prototype.

## Phase 0: Engineering Foundation

Status: in progress

- Create repository structure.
- Define contributor and agent standards.
- Establish GitHub workflow.
- Document architecture boundaries.
- Record technology recommendations.
- Define development master plan.

## Phase 1: Product Specification

- Define learner personas and primary workflows.
- Write product requirements for Instant Recall Cards.
- Write product requirements for NBME-style questions.
- Define content object schemas at a conceptual level.
- Define medical review workflow.
- Create initial design principles and low-fidelity flows.

## Phase 2: Technical Architecture Decisions

- Decide application framework.
- Decide database and ORM strategy.
- Decide authentication approach.
- Decide offline and PWA strategy.
- Decide search architecture.
- Decide analytics and observability approach.
- Create ADRs for approved choices.

## Phase 3: Design System Foundation

- Define typography, color, spacing, iconography, and layout rules.
- Build component inventory.
- Create accessibility acceptance standards.
- Prototype major workflows before production code.

## Phase 4: MVP Application Shell

- Create app scaffold.
- Add routing, layout shell, design tokens, and test framework.
- Add authentication only after its ADR is approved.
- Add development, preview, and deployment workflows.

## Phase 5: Learning Object Core

- Implement Instant Recall Card models and editor workflows.
- Implement review sessions.
- Implement mastery states.
- Add search and filtering.
- Add QA and accessibility coverage.

## Phase 6: Question System

- Implement NBME-style question authoring.
- Implement distractor and explanation structure.
- Implement review and approval states.
- Add citations and medical accuracy checks.

## Phase 7: Retention, Analytics, And Scale

- Implement spaced repetition.
- Add learner dashboards.
- Add offline-first review behavior if approved.
- Add performance instrumentation.
- Add content quality analytics.
