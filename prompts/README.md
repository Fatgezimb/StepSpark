# Prompts

Reusable prompts for StepSpark should live here.

## Prompt Requirements

Each prompt should document:

- Purpose
- Inputs
- Expected output
- Quality checks
- Known failure modes
- Required human review

## Guardrails

- Do not use prompts to generate final medical content without review.
- Do not include secrets, learner data, or credentials.
- Prefer structured outputs that can be validated.
