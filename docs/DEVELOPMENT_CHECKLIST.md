# Development Checklist

Use this checklist before opening or merging a pull request.

## Planning

- [ ] Issue or task exists.
- [ ] Scope is clear.
- [ ] Relevant documentation has been read.
- [ ] ADR exists for major technical choices.

## Implementation

- [ ] Change is focused.
- [ ] Naming matches product language.
- [ ] Domain logic is separated from UI where applicable.
- [ ] No unrelated refactors are included.
- [ ] No secrets or private data are committed.

## Quality

- [ ] Tests were added or updated when behavior changed.
- [ ] Accessibility impact was considered.
- [ ] Performance impact was considered.
- [ ] Security impact was considered.
- [ ] Medical education impact was considered.

## Documentation

- [ ] Relevant docs were updated.
- [ ] Decisions were recorded.
- [ ] Changelog was updated if user-facing behavior changed.

## Review

- [ ] Pull request template is complete.
- [ ] Risks are documented.
- [ ] Reviewers are assigned based on affected areas.
