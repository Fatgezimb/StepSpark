# Documentation Standards

## Purpose

Documentation should help future contributors make correct decisions without relying on chat history.

## Rules

- Keep documentation close to the workflow it supports.
- Update docs in the same pull request as related changes.
- Prefer clear examples over broad principles alone.
- Link issues, pull requests, and decisions.
- Mark speculative ideas as proposals.
- Remove outdated claims when they are replaced.

## Required Documentation Updates

Update docs when changing:

- Architecture
- User workflows
- Medical content policy
- Accessibility behavior
- Security behavior
- Data models
- Development commands
- Deployment or release process
